## Decisions locked this session

### D1 — Product direction
Plan L1 (multi-user SaaS on Next.js + Supabase) is the direction.
Plan L2 (single-user Vite + Cloudflare assistant) is abandoned.
The `archive/vite-shell/` folder remains preserved for design reference only.

### D2 — Audience breadth
Broaden from "DJ-first" to **any service provider + client booking**:
photographers, videographers, fitness trainers, beauty/wellness, tutors,
appointment-based services generally, in addition to entertainment.

This is a substantive change to `docs/source-of-truth.md` — multiple
locked truths currently say "DJ-first" or "DJ / Manager Lite only."

### D3 — Nightlife-day as a workspace setting
Nightlife-day / cross-midnight / after-hours / 12pm-6am service-day
logic survives D2 but becomes a **workspace-level setting**, not a
global default:

- `service_day_mode` on workspace: `"standard"` (midnight cutoff) or
  `"nightlife"` (6am cutoff, configurable)
- Onboarding asks the workspace creator which mode fits their use
- `schedule_commitments`, booking rendering, and conflict detection
  respect the workspace's mode
- Owner's personal workspace defaults to `"nightlife"`

This honors the reason the product exists for the owner (missing a
Friday-night after-hours gig because it rendered on Saturday) without
forcing daytime-service users to deal with the complexity.
